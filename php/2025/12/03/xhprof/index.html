XHProf性能分析工具。

发表于 2025-12-03 | 更新于 | 分类于 php

XHProf 最初由 Facebook 开源，曾是 PHP 性能分析的标杆工具；如今在社区的持续维护下，已实现对 PHP 8.4 的完整支持，凭借其低开销采样、多维度指标采集（时间、内存、调用栈）和可视化调用图能力，依然是 PHP 开发者在开发调试与生产诊断中进行性能瓶颈定位的高效利器。

XHProf2.3.10报告表头

英文表头 中文解释 详细说明与重要性
Function Name 函数名称 被调用的函数或方法的完整名称。例如 main(), mysql_query, UserController::index()。这是识别代码位置的关键。
Calls 调用次数 该函数在此次请求中被调用的总次数。
重要性：如果一个函数被调用了成千上万次（例如循环内），即使单次很快，累积起来也可能成为性能瓶颈。
Calls% 调用次数占比 该函数的调用次数占总调用次数的百分比。
用途：帮助识别哪些函数是程序的主要执行路径。
Incl. Wall Time (microsec) 总耗时 (微秒)
(或: 包含实际时间)		 核心指标。指函数从开始到结束的总耗时，包括其内部调用的所有子函数的时间。
单位：微秒 (1秒 = 1,000,000 微秒)。
重要性：用于定位哪个大的功能模块最耗时。
IWall% 总耗时占比
(或: 包含实际时间占比)		 该函数的包含时间占总包含时间的百分比。
重要性：直观地看出每个函数对总响应时间的“贡献”比例。main() 函数永远是 100%。
Excl. Wall Time (microsec) 自身耗时 (微秒)
(或: 独占实际时间)		 另一个核心指标。指函数自身的执行时间，不包括其内部调用的任何子函数的时间。
重要性：这是优化的关键！如果这个值很高，说明函数自身的逻辑（算法、循环、计算）需要优化。
EWall% 自身耗时占比
(或: 独占实际时间占比)		 该函数的独占时间占总时间的百分比。
用途：辅助判断函数自身逻辑的耗时占比。但通常我们更关注绝对值。
Incl. CPU (microsecs) 总CPU时间 (微秒) 类似于 总耗时，但是统计的是CPU实际工作的时间，排除了等待时间（如I/O、网络、sleep）。
重要性：如果 总耗时 很高但 总CPU时间 很低，说明瓶颈可能在I/O或外部服务（数据库、API调用）。
Excl. CPU (microsecs) 自身CPU时间 (微秒) 类似于 自身耗时，是函数自身消耗的纯CPU时间。
Incl. MemUse (bytes) 包含内存占用 (字节) 函数执行过程中分配的内存总量，包括其子函数分配的内存。
Excl. MemUse (bytes) 独占内存占用 (字节) 函数自身直接分配的内存量，不包括子函数。
重要性：用于发现内存泄漏或大对象分配的热点。
Incl. PeakMemUse (bytes) 包含峰值内存 (字节) 在函数执行期间，整个进程达到的内存使用峰值，包括子函数的影响。
Excl. PeakMemUse (bytes) 独占峰值内存 (字节) 在函数自身执行栈帧内达到的内存峰值。

<?php
xhprof_enable(XHPROF_FLAGS_CPU + XHPROF_FLAGS_MEMORY);

for ($i = 0; $i <= 1000; $i++) {
    $a = $i * $i;
}

$xhprof_data = xhprof_disable();

$XHPROF_ROOT = "/tools/xhprof/";
include_once $XHPROF_ROOT . "/xhprof_lib/utils/xhprof_lib.php";
include_once $XHPROF_ROOT . "/xhprof_lib/utils/xhprof_runs.php";

$xhprof_runs = new XHProfRuns_Default();
$run_id = $xhprof_runs->save_run($xhprof_data, "xhprof_testing");

echo "http://localhost/xhprof/xhprof_html/index.php?run={$run_id}&source=xhprof_testing\n";

?>
