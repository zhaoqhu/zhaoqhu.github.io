PHP面试参考题目：
1. ThinkPHP的中间件的作用是什么?
中间件的作用是处理HTTP请求到达控制器之前或者返回给客户端信息之前的处理，比如RBAC权限认证、JWT校验、日志记录、压缩响应头、添加Http头(Access-Control-Allow-Origin等)
2. 如何解决跨域（一定要说JWT）
情况一：项目依赖session时
如果后端项目是多台服务器集群，首先将session存储在redis中，然后通过redis的session共享。
然后通过nginx反向代理，将/api/xxx 映射到后端服务器的 /api/xxx。

情况二：项目不依赖session时，可以使用JWT，JWT是一种跨域认证的解决方案。
3. mysql的四大特性
ACID：
原子性：事务中的所有sql操作要么全部完成，要么全部不执行。
一致性：事务执行前后，数据库必须保持一致性状态。
隔离性：多个事务并发执行时，彼此之间互不干扰，通过锁机制、MVCC（多版本并发控制）实现
持久性：事务一旦提交，对数据库的修改就是永久性的

4. PHP的try catch捕获异常中的错误原理
可捕获：通过throw语句主动抛出的Exception对象和Throwable对象。
5. TP接口遇到报错 你要怎么处理 处理步骤
找到系统的运行日志，使用tail -f 查看日志的信息，然后分析错误信息，然后定位问题，然后解决。
6. 如何使用非阻塞 IO 处理高并发场景？
传统的php-fpm项目是同步阻塞IO，如果接口中有多个查询操作，会导致大量请求阻塞，造成服务器负载过高。
如果使用hyperf（协程+IO多路复用）这样的高性能异步非阻塞框架，可以异步非阻塞处理查询商品、更新库存、创建订单等操作，从而提升接口吞吐量。
7. 有关php的函数和方法有什么区别
函数定义方式不依赖类，方法定义方式依赖类，函数可以直接调用，方法需要通过对象来调用。
8. 服务器分布 高并发怎么处理？
静态资源使用CDN加速，
后端服务器使用LVS/Nginx做负载均衡
高并发使用异步消息队列处理，hyperf协程框架处理高并发。

用户输入域名
    ↓
DNS解析（可能返回CDN的IP，也可能返回LB的IP）
    ↓
① 静态资源请求 → CDN边缘节点（命中则直接返回，未命中回源到源站）
② API请求    → 负载均衡器（CLB/Nginx）→ 后端某台服务
    ↓
浏览器渲染页面


9. 线上出现问题如何快速定位？
根据页面返回的错误信息，如果接口返回504错误则认为是php-fpm执行超时，如果是502错误则认为是php-fpm挂掉了，如果是500错误则认为是代码错误。然后根据thinkphp的运行日志，查看错误信息，然后定位问题。
10. ThinkPHP鉴权
用户通过接口获取JWT，然后通过 http header bearer token 使用中间件校验JWT鉴权，然后根据用户身份做RBAC鉴权。

乐观锁（Optimistic Lock）
1️⃣ 是什么？
我不锁数据，更新时看有没有被别人改过。
✅ 通过 版本号 / 时间戳​ 实现。
$res = Db::name('product')
    ->where('id', 1)
    ->where('version', $product['version'])
    ->update([
        'stock' => $product['stock'] - 1,
        'version' => $product['version'] + 1
    ]);

if ($res === 0) {
    // 更新失败，说明被别人改过了
    return json(['code' => 409, 'msg' => '请重试']);
}
悲观锁，在操作数据前，先锁定这条数据。依赖事务和where条件必须是命中主键索引或唯一索引，并且用=条件这个是锁的行为是行锁。
如果where条件是命中主键索引或唯一索引，但是使用非=条件，那么锁的行为是间隙锁。
如果where条件命中的是普通索引，那么锁的行为是间隙锁。
如果where条件没有命中索引，那么锁的行为是表锁。
Db::startTrans();
try {
    $product = Db::name('product')
        ->lock(true)
        ->find(1);

    if ($product['stock'] > 0) {
        Db::name('product')
            ->where('id', 1)
            ->dec('stock')
            ->update();
    }

    Db::commit(); // 提交时才释放锁
} catch (\Exception $e) {
    Db::rollback();
}

13. php常用设计模式也没说到细节


13.1 单例模式（Singleton）—— 最常用
对象只想创建一次，节省资源。
场景：数据库连接、Redis 连接、日志对象。

13.2 工厂模式（Factory）--解耦创建逻辑
创建逻辑复杂，不方便维护。
场景：支付（支付宝 / 微信）
支付方式
interface PayInterface {
    public function pay();
}

class AliPay implements PayInterface {
    public function pay() {
        echo "支付宝支付";
    }
}

class WechatPay implements PayInterface {
    public function pay() {
        echo "微信支付";
    }
}
工厂类
class PayFactory {
    public static function create($type): PayInterface {
        switch ($type) {
            case 'ali':
                return new AliPay();
            case 'wechat':
                return new WechatPay();
            default:
                throw new Exception("不支持的支付方式");
        }
    }
}
使用
$pay = PayFactory::create('ali');
$pay->pay();

13.3 注册器模式（Registry）—— 全局存取

场景：全局配置、全局用户信息。
class Registry {
    private static $data = [];

    public static function set($key, $value) {
        self::$data[$key] = $value;
    }

    public static function get($key) {
        return self::$data[$key] ?? null;
    }
}
使用 
Registry::set('uid', 1001);
$uid = Registry::get('uid');
13.4 观察者模式（Observer）—— 解耦事件
一个动作触发多个行为，不修改原代码，扩展新行为
事件
class UserRegister {
    private $observers = [];

    public function attach($observer) {
        $this->observers[] = $observer;
    }

    public function notify() {
        foreach ($this->observers as $observer) {
            $observer->update();
        }
    }
}
观察者
class SendEmail {
    public function update() {
        echo "发送邮件\n";
    }
}

class AddPoints {
    public function update() {
        echo "增加积分\n";
    }
}
使用
$register = new UserRegister();
$register->attach(new SendEmail());
$register->attach(new AddPoints());
$register->notify();
