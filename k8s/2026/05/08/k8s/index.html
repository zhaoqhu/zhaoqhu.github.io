单机环境安装

首先，我们开始准备Kubemctes的安装和相关镜像下载，本书建议采用VirtualBox或者’Mware Workstation在本机虚拟一个64位的CentOS7虚拟机作为学习环境，虚拟机采用NAT的网络模式以便能够连接外网，然后按照以下步骤快速安装Kubermnetes。

  • (1)关闭CentOS自带的防火墙服务:
1
2
# systemctl disable firewalld
# systemctl stop firewalld
  • (2)安装etcd和Kuberetes软件(会自动安装Docker软件):
1
# yum install -y etcd kubernetes
  • (3)按顺序启动所有的服务:
1
2
3
4
5
6
7
# systemctl start etcd
# systemctl start docker
# systemctl start kube-apiserver
# systemctl start kube-controller-manager
# systemctl start kube-scheduler
# systemctl start kubelet
# systemctl start kube-proxy

至此，一个单机版的Kubemetes集群环境就安装启动完成了。 接下来，我们可以在这个单机版的Kubemetes集群中上手练习了。

集群环境安装

Kubernetes 集群至少需要两个节点：一个 Master 节点（控制平面）和一个或多个 Node 节点（工作节点）。下面介绍如何搭建一个多节点的 Kubernetes 集群。

前置准备

在开始之前，确保所有节点满足以下要求：

  • 操作系统：CentOS 7 (64 位) 或兼容的 Linux 发行版
  • 内存：Master 节点至少 2GB，Node 节点至少 1GB
  • 网络：所有节点之间可以互相通信

所有节点通用配置（两种方式都需要）

  • (1) 配置主机名和 hosts 解析：

在 Master 节点上：

1
# hostnamectl set-hostname master

在 Node 节点上：

1
# hostnamectl set-hostname node1

在所有节点的 /etc/hosts 文件中添加：

1
2
3
<MASTER_IP> master
<NODE1_IP> node1
<NODE2_IP> node2
  • (2) 关闭 SELinux：
1
2
# setenforce 0
# sed -i 's/^SELINUX=enforcing$/SELINUX=disabled/' /etc/selinux/config
  • (3) 关闭防火墙：
1
2
# systemctl stop firewalld
# systemctl disable firewalld
  • (4) 关闭 Swap 分区：
1
2
1
2
# swapoff -a
# sed -i '/ swap / s/^/#/' /etc/fstab
  • (5) 配置内核参数：

Master 节点安装

在 Master 节点上需要执行以下步骤：

  • (1) 安装 etcd、Kubernetes 和 Docker：
1
# yum install -y etcd kubernetes docker
  • (2) 配置 kube-apiserver：

编辑 /etc/kubernetes/apiserver 文件，设置 API 服务器的绑定地址和端口：

1
2
3
4
KUBE_API_ADDRESS="--insecure-bind-address=0.0.0.0"
KUBE_API_PORT="--port=8080"
KUBE_MASTER_ADDRESS="--advertise-address=<MASTER_IP>"
KUBE_ADMISSION_CONTROL="--admission-control=NamespaceLifecycle,LimitRanger,ServiceAccount,DefaultStorageClass,ResourceQuota"
  • (3) 配置 etcd 集群地址（如果是多 Master 环境）：

编辑 /etc/kubernetes/config 文件：

1
KUBE_ETCD_SERVERS="--etcd-servers=http://<MASTER_IP>:2379"
  • (4) 配置 kube-controller-manager 和 kube-scheduler（可选）：

默认情况下它们会自动连接到本地 apiserver，如有需要可编辑 /etc/kubernetes/controller-manager 和 /etc/kubernetes/scheduler 进行自定义配置。

  • (5) 按顺序启动 Master 节点的服务：
1
2
3
4
5
6
# systemctl start etcd
# systemctl start docker
# systemctl start kube-apiserver
# systemctl start kube-controller-manager
# systemctl start kube-scheduler
# systemctl enable etcd docker kube-apiserver kube-controller-manager kube-scheduler

Node 节点安装

在每个 Node 节点上需要执行以下步骤：

  • (1) 安装 Kubernetes 和 Docker：
1
# yum install -y kubernetes docker
  • (2) 配置 kubelet：

编辑 /etc/kubernetes/kubelet 文件，设置 API 服务器地址：

1
2
3
KUBELET_ADDRESS="--address=0.0.0.0"
KUBELET_PORT="--port=10250"
KUBELET_API_SERVER="--api-servers=http://<MASTER_IP>:8080"
  • (3) 配置 kube-proxy：

编辑 /etc/kubernetes/proxy 文件，设置 Master 的 API 服务器地址：

1
2
KUBE_PROXY_ADDRESS="--bind-address=0.0.0.0"
KUBE_PROXY_API_SERVER="--api-servers=http://<MASTER_IP>:8080"
  • (4) 关闭 Swap 分区（kubeadm 要求）：
1
2
1
2
# swapoff -a
# sed -i '/ swap / s/^/#/' /etc/fstab
  • (5) 启动 Node 节点的服务：
1
2
3
4
# systemctl start docker
# systemctl start kubelet
# systemctl start kube-proxy
# systemctl enable docker kubelet kube-proxy
  • (6) 重启服务使配置生效：

如果节点没有自动加入集群，尝试重启服务：

1
2
# systemctl restart kubelet
# systemctl restart kube-proxy

验证集群状态

在 Master 节点上执行以下命令验证节点是否加入集群：

1
1
# kubectl get nodes

如果配置正确，应该能看到所有 Node 节点都处于 Ready 状态。

配置节点角色和标签

默认情况下，Master 节点也会承担工作负载。如果你希望 Master 节点只运行控制平面组件，可以设置污点：

1
# kubectl taint nodes <master-node-name> node-role.kubernetes.io/master=:NoSchedule

为 Node 节点添加标签以便调度：

1
# kubectl label nodes <node-name> node-role.kubernetes.io/worker=worker

再次查看所有节点状态：

1
# kubectl get nodes --show-labels

至此，一个多节点的 Kubernetes 集群环境就搭建完成了。

两种方式对比

特性 yum 安装（配置文件方式） kubeadm（命令行方式）
Kubernetes 版本 较老（1.x 早期版本） 最新版本
配置方式 修改配置文件 命令行交互
Node 加入方式 配置 kubelet 和 proxy 指向 Master 执行 kubeadm join 命令
认证方式 无加密认证 Token + 证书哈希安全认证
推荐场景 学习老版本架构 生产环境部署

常见问题

1. Node 节点一直处于 NotReady 状态

  • 检查网络是否通畅
  • 检查防火墙是否关闭
  • 检查 Docker 和 kubelet 服务是否运行
  • 查看日志：journalctl -u kubelet -f

2. kubeadm join 失败

  • 确保 Master 节点初始化时记录的 token 和 hash 正确
  • 确保 6443 端口可访问
  • 使用 kubeadm reset -f 重置后重新加入

3. Pod 无法跨节点通信

  • 确保安装了网络插件（Flannel、Calico 等）
  • 检查内核参数是否正确配置

    • name: https port: 443 targetPort: 443 nodePort: 30443 ```
