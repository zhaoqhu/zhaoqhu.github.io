在不修改原函数代码的前提下，动态扩展函数功能的工具。使用场景：日志记录、权限校验、性能监控等。

from functools import wraps

def log_decorator(func):
    @wraps(func)  # 保留原函数元信息
    def wrapper(*args, **kwargs):
        print(f"调用 {func.__name__}")
        result = func(*args, **kwargs)
        print(f"{func.__name__} 执行完毕")
        return result
    return wrapper

@log_decorator
def greet(name):
    print(f"Hello, {name}!")

greet("Alice")

  • 3.生成器Generator是什么？如何使用？

定义：通过 yield关键字逐次生成值的函数，支持惰性计算（按需生成）。 优势：节省内存（适合大数据流）、支持状态保持。

# 生成器函数
def count_up_to(n):
    i = 0
    while i < n:
        yield i
        i += 1

gen = count_up_to(3)
print(list(gen))  # [0, 1, 2]

# 生成器表达式
squares = (x**2 for x in range(5))
print(list(squares))  # [0, 1, 4, 9, 16]

Uvicorn 运行 FastAPI 异步框架通过 多进程模型​ 和 协程调度​ 突破 Python GIL 限制，实现高并发。其核心机制如下
1. GIL 限制的突破
多进程并行：
Uvicorn 通过启动多个 Worker 进程（--workers参数），每个进程拥有独立的 Python 解释器和 GIL，可并行处理 CPU 密集型任务（如模型推理），充分利用多核 CPU。
异步协程：
在单线程内，协程通过 async/await实现非阻塞 I/O 操作（如数据库查询），避免线程阻塞，提升 I/O 密集型任务的吞吐量

2.
1. CPU 密集型任务的局限性
纯异步无法突破 GIL：若任务涉及大量 CPU 计算（如图像处理），即使使用协程，仍受限于单线程的 GIL。
解决方案：
多进程：通过 multiprocessing或 Uvicorn 多 Worker 分配任务到不同进程。
C 扩展：将计算密集型代码用 C/C++ 编写，绕过 GIL（如 NumPy、Cython）。
2. 混合并发模型

  • 10.__slots__是什么？如何使用？

定义：限制类的实例属性，减少内存占用。 语法：在类中定义 slots = (‘attr1’, ‘attr2’) 优势： 阻止动态添加未声明的属性。 内存优化（避免为每个实例创建 dict）。

class Point:
    __slots__ = ('x', 'y')
    
    def __init__(self, x, y):
        self.x = x
        self.y = y

p = Point(1, 2)
p.z = 3  # AttributeError: 'Point' object has no attribute 'z'
